## 🚀 验证步骤（按优先级）

### 优先级1️⃣ - 立即验证

#### t13_shift_boundary - 移位边界
```
关键验证:
✓ sll x9, x1, 31 → x9 = 0x80000000 (1左移31位)
✓ srl x11, x2, 31 → x11 = 0x00000001 (全1右移31位，逻辑)
✓ sra x13, x2, 31 → x13 = 0xFFFFFFFF (全1右移31位，算术补1)
✓ sra x14, x4, 31 → x14 = 0x00000000 (正数右移31位，补0)

如果所有结果正确 ✓ → 移位指令边界值完美
如果结果错误 ✗ → 检查CPU的移位单元逻辑
```

#### t14_add_negative - 负数算术
```
关键验证:
✓ x8 = 0xFFFFFFFD (负1 + 负2 = 负3)
✓ x9 = 0x00000004 (负1 + 5 = 4)
✓ x10 = 0xFFFFFFFA (负1 - 5 = 负6)
✓ x12 = 0x00000011 (16 - 负1 = 17)
✓ x13 = 0x00000000 (负1 + 1 = 0)

如果所有结果正确 ✓ → 算术单元符号处理完美
如果结果错误 ✗ → 检查ALU的加法/减法逻辑
```

### 优先级2️⃣ - 补充验证

#### t15_bne_false - BNE条件反向
```
关键验证:
✓ x1 == x2 时bne不跳转 → x5 = 0xAA, x6 = 0x11 (都执行)
✓ x3 == x4 时bne不跳转 → x7 = 0x77, x8 = 0x88 (都执行)

如果所有结果正确 ✓ → BNE逻辑完整
如果结果错误 ✗ → 检查CPU的条件判断逻辑
```

### 优先级3️⃣ - 深化验证（可选）

#### t16_lw_negative_extend - 负偏移读取
```
验证CPU可正确处理更多负偏移场景
```

## 🔍 如何调试失败的测试

### 1️⃣ 确认.s文件中的预期值
```
打开 t##_*.s → 查看注释中的期望结果
例如: "x8 = 0xFFFFFFFD (负1 + 负2 = 负3)"
```

### 2️⃣ 在模拟器中逐步执行
```
模拟器 → 加载 t##_*.hex
单步执行 → 观察每条指令后的寄存器变化
```

### 3️⃣ 对比实际值与预期值
```
实际: x8 = 0x00000000 ✗
预期: x8 = 0xFFFFFFFD ✓
差异: 加法结果错误 → 检查ALU
```

### 4️⃣ 定位硬件模块
```
如果移位指令出错 → 检查 shifter 模块
如果加法出错 → 检查 adder 模块
如果跳转出错 → 检查 PC 计算逻辑
```
